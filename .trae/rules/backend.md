## 后端规范
1. 技术栈 
   - 运行环境: Node.js 22.x 以上
   - 框架: Express.js 5.2.x
   - 数据库: Prisma, SQLite
   - 数据校验: Zod 4.4.x
   - 日志: Pino 10.3.x
   - 语言: TypeScript 6.x
2. 代码规范
   - 代码格式化: eslint
   - 代码注释: JSDoc 格式, 提供详细的函数、类、模块等的注释, 包括参数、返回值、异常等信息
3. 代码质量
   - 代码质量检查: eslint
4. 架构设计
   - 采用分层架构: 控制器层, 业务层/服务层, 适配/集成层, 数据访问层
   - 控制器层: 负责接收客户端请求, 调用业务层接口
   - 业务层/服务层: 负责处理业务逻辑，调用数据访问层和适配/集成层接口
   - 适配/集成层: 负责与外部系统交互, 如第三方服务等
   - 数据访问层: 负责与数据库交互, 提供数据库操作接口
5. 数据库设计
   - 数据库采用 SQLite, 提供简单、快速的数据库操作
   - 数据库表结构设计: 采用 Prisma 定义数据库表结构, 确保数据库表结构的正确性和一致性
   - 数据库索引设计: 采用 Prisma 定义数据库索引, 确保数据库索引的正确性和一致性
   - 数据库事务设计: 采用 Prisma 定义数据库事务, 确保数据的一致性和事务性
6. 数据校验设计
   - 数据校验采用 Zod 4.4.x, 提供类型安全的校验
   - 数据校验规则设计: 确保数据的校验失败时, 能够及时返回错误信息
7. API 设计
   - API 采用 RESTful 风格, 提供统一的接口规范
   - 接口认证设计: 采用 JWT 认证, 确保接口的安全性
   - 接口返回设计: 返回 JSON 格式数据, 包含状态码、消息、数据字段
   - 接口错误码设计: 采用 HTTP 状态码, 确保接口的可操作性和可维护性
   - 接口日志设计: 采用 Pino 10.3.x, 提供详细的日志记录, 包括请求日志、响应日志等, 确保接口的可监控性和可调试性
